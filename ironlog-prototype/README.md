# IronLog — UI-прототип (handoff)

Интерактивный макет из бандла Claude Design: экраны дашборда, библиотеки упражнений, конструктора, активной тренировки, прогресса, профиля и т.д. Вёрстка и логика на **React 18** в браузере через **@babel/standalone** (без сборки).

## Файлы

| Путь | Назначение |
|------|------------|
| `project/IronLog.html` | Точка входа: оболочка «телефона», навигация, панель Tweaks |
| `project/ironlog-data.jsx` | Статические данные (упражнения, шаблоны, графики) |
| `project/ironlog-common.jsx` | Тема, статус-бар, нижняя навигация, общие виджеты |
| `project/ironlog-screens-a.jsx` | Экраны: дашборд, упражнения, карточка, конструктор |
| `project/ironlog-screens-b.jsx` | Экраны: активная тренировка, прогресс, профиль, журнал, программа |
| `project/tweaks-panel.jsx` | Отдельный фрагмент панели (в HTML используется встроенная версия) |

Исходный архив в корне репозитория: **`IronLog-handoff.zip` — это и есть экспорт `ironlog.zip`** из Claude Design (одно и то же handoff-бандло).

## Запуск

Откройте **`project/IronLog.html` только через локальный HTTP-сервер** (из папки `project`), иначе браузер не загрузит внешние `.jsx` для Babel.

Из корня репозитория:

```bash
cd ironlog-prototype/project
python -m http.server 8765
```

Затем в браузере: [http://127.0.0.1:8765/IronLog.html](http://127.0.0.1:8765/IronLog.html)

Нужен доступ в интернет для CDN: `unpkg.com` (React, ReactDOM, Babel).

## Дальнейшая интеграция

По README handoff: переносить визуал **пиксель-в-пиксель** в целевой стек (например, Next.js 14 в каталоге `web/`), не копируя слепо внутреннюю структуру прототипа. Цвета и смысл экранов совпадают с `CLAUDE.md` (accent `#7c6ef2`, тёмная база и т.д.).
